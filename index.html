valtors

i run an open source org called valtors. four tools. all MIT licensed. zero telemetry. your data stays on your machine. no pricing tiers. no enterprise plan. no bullshit.

vault - sandbox for ai agents. filesystem overlay, network proxy, mcp gate, prompt injection scanner, audit log. run your agent in it. it can't destroy your machine. go, single binary, 33 tests.

relay - one MCP server, 40 tools, one Go binary. files, images, PDFs, web fetch, search, git, system commands. other people install 5 servers to do what this does alone. 10 stars on github. available on npm as userelay.

reflow - responsive toolkit for TypeScript. same API in React, Vue, Svelte, Solid, Qwik, Preact, Angular, and vanilla JS. no hydration mismatch warnings. no typeof window checks. no framework lock-in. 7 stars. available on npm as usereflow.

observer - MCP proxy that logs every tool call your AI agent makes to SQLite. you see what actually happened, not what the agent decided to show you. SSE transport, tool filtering, trace replay. security hardened by an external contributor who found raw payloads leaking. 3 stars.

mcprobe - security scanner for MCP servers. finds prompt injection in tool descriptions, tool shadowing, resource exposure, baseline drift. 18 injection patterns. zero dependencies. single Go binary. this is the one i care about most.

the code is silent. the output speaks.

every string in every valtors tool has a voice. blunt. lowercase. no apologies. when mcprobe finds a vulnerability, it says "your server is probably vulnerable. you connected it to your agent without checking." when relay can't find a command, it says "that's not a command. run 'relay help' if you're lost." no other developer tools do this. that's the point.

contributions

i contribute to other people's projects. not for badges, not for clout. because building things in isolation teaches you less than building things next to people who are better than you.

ClickHouse - two merged PRs. vector search optimization with LIMIT ... WITH TIES, then a follow-up fixing the edge case rschu1ze found. ClickHouse test framework is strict about exact output matching. one extra newline breaks every sanitizer build. i learned that the hard way. twice.

labstack/echo - added llms.txt and llms-full.txt to their docs. machine-readable documentation so AI agents can understand a project without scraping the site. maintainer aldas merged it and said "hope it helps people and their LLMs." he gets it.

steel-browser - fixed a session leak. user data directories weren't cleaned up when sessions ended. your next session could see the previous session's cookies. same class of bug as MCP tool shadowing. state bleeds. nobody notices until someone reads the logs.

agent-orchestrator - multiple PRs. lifecycle management, session guard, reaper improvements. the repo itself is an MCP-based system for orchestrating AI coding agents. building the thing that controls agents teaches you what agents read. they read everything. including tool descriptions. including the ones that say "ignore all previous instructions."

microsoft/autogen - contributed a recipe for multi-agent workflows. CLA signed. 65 tests passing. blocked on fork CI approval because microsoft's CI doesn't run on forks without manual approval. that's the process. you wait.

17 PRs merged. 10 open. every one of them real work. no link dumps, no typo fixes for green squares.

the blog

i wrote a thing about MCP security. it's about how every MCP server you connect to your AI agent is code you didn't write, running on your machine, reading your files, driven by a model that follows instructions. including instructions written by someone else. including the ones that say "ignore all previous instructions."

Simon Willison, who coined the term "prompt injection," said the problem has been known for two and a half years and we still don't have convincing mitigations. Invariant Labs proved the attack against Cursor. Snyk disclosed real CVEs. OWASP formalized it. nobody checked the servers. that's what the blog is about.
